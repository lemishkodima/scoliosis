# Wow Design Redesign Plan

## Мета

Перетворити поточний прототип із "сайту з гарним відео" на преміальний бренд-досвід для професійної асоціації: сильний перший екран, кінематографічне завантаження, глибина при скролі, чисті фірмові мотиви та відчуття статусності членства.

## Основні принципи

- Hero має працювати як сцена, а не як звичайний блок із фоновим відео.
- Loader має відкривати сайт через планки/вікно, швидко і впевнено.
- Перший контентний блок має накладатися поверх hero як преміальна панель.
- Анімації мають бути архітектурними: маски, reveal, depth, control, без хаотичних рухів.
- Логотип із круговим текстом використовується як фірмовий системний мотив.
- Курсор, header і secondary effects не повинні перебивати головний фокус.

## Етапи

### Етап 0. Image Performance Hotfix

Статус: done

Задачі:
- [x] Конвертувати важкі PNG-фото у WebP.
- [x] Перемкнути HTML на WebP-версії.
- [x] Прибрати PNG-оригінали з git, щоб деплой не тягнув сотні мегабайт.
- [x] Додати `width`/`height` для зображень, щоб зменшити layout shift.

Результат:
- Важкі фото зменшені з 16-28 MB до приблизно 56-394 KB.
- Логотипи зменшені до 12-44 KB.
- Сайт більше не має активних посилань на PNG-зображення.

### Етап 1. Loader + Hero + First Overlay

Статус: done

Задачі:
- [x] Скоротити loader і зробити появу більш premium.
- [x] Посилити ефект планок, які відкривають контент як вікно.
- [x] Перебудувати hero у cinematic scene з глибиною відео.
- [x] Об'єднати CTA, lead і напрями в акуратний control dock.
- [x] Зробити intro-блок виразним overlay-шаром поверх hero.

Результат:
- Loader тепер коротший, з більш виразними планками та швидшим відкриттям контенту.
- Hero отримав depth-площини, сильніші маски, light sweep перед повтором відео.
- CTA, lead і напрями об'єднані в один glass control dock.
- Intro-блок піднятий поверх hero як окремий overlay-шар.

Статус: done

- [x] Зробити header максимально легким на старті.
- [x] Посилити glass-state тільки після скролу.
- [x] Зменшити конфлікт курсору з текстом.

### Етап 3. Section Animation System

Статус: done

Задачі:
- [x] Замінити простий fade-up на mask/clip reveal.
- [x] Додати stagger для карток.
- [x] Додати reveal для фото через вертикальну маску.
- [x] Зробити секції більш шаровими, щоб скрол відчувався як рух камери.

Результат:
- Reveal-система тепер застосовується до section heading, media-блоків, галереї, кроків і форми.
- Фото проявляються через vertical mask із легким scale-down.
- Картки й секції мають stagger через `--reveal-index`.

### Етап 4. Image Art Direction

Статус: done

Задачі:
- [x] Відібрати зображення, які напряму пов'язані зі спиною, сколіозом, терапією, ортезуванням.
- [x] Уніфікувати crop, contrast, blue medical grading.
- [x] Прибрати відчуття випадкового набору stock-зображень.

Результат:
- Для mission, audience і gallery додано єдиний холодний medical grading.
- Фото отримали overlay-шари, контрольований contrast/saturation і більш керовані crop-и.
- Медіа-блоки стали ближчими до editorial medical identity, а не до випадкового набору картинок.

### Етап 5. Final Visual QA

Статус: done

Задачі:
- [x] Перевірити desktop hero на 1280x720 viewport.
- [x] Перевірити mobile hero на 390x844 viewport.
- [x] Перевірити плавність скролу на рівні DOM/console без помилок.
