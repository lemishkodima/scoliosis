## Форма заявки

### Мінімальний набір полів

- `fullName` - імʼя та прізвище.
- `email` - електронна пошта.
- `phone` - телефон.
- `city` - місто.
- `specialization` - спеціалізація.
- `workplace` - місце роботи або організація.
- `memberType` - категорія спеціаліста.
- `experience` - досвід роботи зі сколіозом.
- `message` - додатковий коментар.
- `consent` - згода на обробку персональних даних.

### Категорії спеціаліста

- Лікар.
- Фізичний терапевт.
- Реабілітолог.
- Ортезист.
- Викладач.
- Студент.
- Інше.

### Валідація

- Імʼя: обовʼязкове.
- Email: обовʼязковий, валідний формат.
- Телефон: обовʼязковий.
- Спеціалізація: обовʼязкова.
- Згода: обовʼязкова.

## Інтеграції заявки

Дані мають надходити у три канали:

1. Google Sheets.
2. Email.
3. Telegram bot.

## Архітектура submit

Frontend надсилає заявку в один backend endpoint:

`POST /api/membership/apply`

Endpoint:

- Валідує дані.
- Нормалізує телефон та email.
- Додає timestamp.
- Відправляє рядок у Google Sheets.
- Відправляє email адміністраторам.
- Відправляє повідомлення в Telegram.
- Повертає frontend успіх або помилку.

Це краще, ніж відправляти дані з браузера напряму в кожен сервіс, бо ключі та токени не потрапляють у frontend.

## Google Sheets

Варіанти реалізації:

### Варіант A. Google Apps Script

Плюси:

- Швидко запустити.
- Добре підходить для MVP.
- Просте додавання рядків у таблицю.

Мінуси:

- Менше контролю над помилками.
- Потрібно акуратно налаштувати доступ.

### Варіант B. Google Sheets API

Плюси:

- Краще для масштабування.
- Більше контролю.
- Нормальна server-to-server інтеграція.

Мінуси:

- Потрібен Google Cloud service account.
- Більше налаштувань.

Рекомендація для MVP: Google Sheets API через serverless endpoint, якщо сайт робиться на Next.js. Apps Script можна використати, якщо потрібен дуже швидкий запуск без складної інфраструктури.

## Email

Рекомендовані сервіси:

- Resend.
- SendGrid.
- Mailgun.

Для MVP достатньо одного transactional email провайдера.

Email має містити:

- Дані заявника.
- Посилання на Google Sheet.
- Timestamp.
- Джерело: `scoliosis.ua`.

- Bot token від BotFather.
- Chat ID групи або адміністратора.
- Backend env variables.

## Захист і приватність

Потрібно додати:

- Checkbox згоди на обробку персональних даних.
- Посилання на політику конфіденційності.
- Rate limiting для endpoint.
- Honeypot field.
- Server-side validation.

На старті можна додати просту сторінку `privacy-policy`, а повну юридичну редакцію підготувати окремо.

## Майбутній roadmap

### Реліз 1

- Лендінг.
- Форма заявки.
- Інтеграції Google Sheets, email, Telegram.
- Базова аналітика.
