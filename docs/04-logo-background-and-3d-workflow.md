# Логотип: вирізання фону та перехід у 3D

## Що зроблено в прототипі

Створено окремий файл:

`assets/images/logo-mark-transparent.webp`

Фон вирізано не простим видаленням білого кольору, а flood-fill методом від країв зображення. Це важливо, бо всередині логотипу є білі елементи: хребет і геометричні лінії. Якщо видаляти всі білі пікселі, ці деталі зникнуть.

У поточному сайті 3D-ефект зроблено як frontend-прототип:

- прозорий PNG логотипу;
- кілька шарів логотипу з різним `translateZ`;
- parallax при русі курсора;
- canvas-частинки та орбітальні лінії;
- світлова підсвітка вздовж хребта.

Це швидкий і легкий спосіб отримати вау-ефект без важкої 3D-моделі на першому MVP.

## Як зробити справжній 3D-обʼєкт

### 1. Підготувати чистий логотип

Потрібні assets:

- прозорий PNG для швидкого використання;
- SVG або AI/EPS для якісного 3D;
- окремі шари: силует, хребет, геометричні лінії, текст.

Якщо є тільки PNG, його потрібно векторизувати.

Рекомендовано:

- Adobe Illustrator: Image Trace -> Expand -> ручне очищення контурів.
- Inkscape: Path -> Trace Bitmap -> ручне очищення.
- Figma: краще тільки для підготовки шарів, не для складної векторизації.

### 2. Розділити логотип на шари

Для 3D краще мати такі групи:

- `body-left` - світла ліва частина силуету.
- `body-right` - темна геометрична частина.
- `spine` - хребет.
- `spine-shadows` - тіні/обʼєм хребта.
- `geometry-lines` - білі геометричні лінії.
- `brand-ring` або `orbit` - додаткове кільце/рухома графіка.

Текстову частину логотипу краще не екструдувати для hero. Її можна залишити в header/footer як плоский brand asset.

### 3. Зробити 3D у Blender

Процес:

1. Імпортувати SVG у Blender.
2. Convert to Mesh або Curve.
3. Дати кожному шару невелику товщину через Extrude.
4. Рознести шари по глибині на 2-8 mm.
5. Хребет зробити найбільш обʼємним елементом.
6. Додати матеріали:
   - світло-блакитний translucent material для лівої частини;
   - темно-синій glossy material для правої частини;
   - білий/молочний material для хребта;
   - emissive accent для spine glow.
7. Додати area light і мʼяку тінь.
8. Експортувати у `.glb`.

### 4. Оптимізувати модель

Перед сайтом:

- зменшити кількість полігонів;
- застосувати transforms;
- обʼєднати дрібні зайві контури;
- перевірити розмір `.glb`;
- бажано тримати hero-модель до 1-2 MB.

### 5. Підключити на сайт

Для майбутнього Next.js/React:

- `three`
- `@react-three/fiber`
- `@react-three/drei`

Для статичного сайту:

- `three.js`
- `GLTFLoader`

Поведінка:

- slow idle rotation;
- cursor parallax;
- scroll-based intro;
- reduced motion fallback;
- mobile fallback на PNG/layered effect.

## Рекомендація

Для MVP залишаємо поточний layered 3D prototype. Для production бажано зробити повноцінну `.glb` модель із SVG/Blender, бо вона буде виглядати значно дорожче, особливо на першому екрані.
